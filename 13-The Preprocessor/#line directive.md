## #line
#line is a preprocessor directive that changes the compiler's internal line numbering and filename for error messages and debugging information. 
It's useful for code generators and preprocessors

```c
#include <stdio.h>

#line 100 "custom_file.c"

int main() {
    printf("This appears as line 100 in custom_file.c\n");
    // If an error occurs here, compiler reports line 100
    return 0;
}
```

## Basic Syntax
```c
#line line_number
#line line_number "filename"
```

## How It Works
### 1. Changing Line Numbers
```c
#include <stdio.h>

int main() {
    printf("Line: %d\n", __LINE__);  // Line: 5 (actual line)
    
    #line 100
    printf("Line: %d\n", __LINE__);  // Line: 100 (changed!)
    
    #line 200
    printf("Line: %d\n", __LINE__);  // Line: 200
    
    return 0;
}
```

### 2. Changing Filename
```c
#include <stdio.h>

int main() {
    printf("File: %s, Line: %d\n", __FILE__, __LINE__);
    
    #line 50 "generated.c"
    printf("File: %s, Line: %d\n", __FILE__, __LINE__);
    
    #line 100 "template.h"
    printf("File: %s, Line: %d\n", __FILE__, __LINE__);
    
    return 0;
}
```

## Common Use Case
### 1. Code Generators
```c
// Generated by a code generator
#line 1 "generated_code.c"

#include <stdio.h>

int main() {
    // This code was generated automatically
    #line 42 "original_template.tmpl"
    printf("This appears as line 42 in original_template.tmpl\n");
    
    #line 100 "generated_code.c"
    printf("Back to generated file\n");
    
    return 0;
}
```
